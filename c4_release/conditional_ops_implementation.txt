==============================================================================
OPERATIONS IMPLEMENTED
==============================================================================

1. SELECT(cond, a, b) → out
   - Ternary conditional: out = cond ? a : b
   - Returns 'a' if cond is true (1), otherwise returns 'b'
   - Uses 4 hidden units to combine both conditional paths
   - Pattern: step(cond >= 0.5) * a + step(cond < 0.5) * b

2. IF_THEN(cond, a) → out
   - Conditional update: out = cond ? a : out
   - Updates output to 'a' if cond is true, leaves unchanged if false
   - Uses 4 hidden units for conditional replacement
   - Pattern: out + step(cond >= 0.5) * (a - out)

==============================================================================
IMPLEMENTATION DETAILS
==============================================================================

SELECT Operation (emit_select):
-------------------------------
Implements: out = step(cond >= 0.5) * a + step(cond < 0.5) * b

Units 0-1: Conditional path for 'a' - step(2*cond >= 1) * a
  Unit 0:
    W_up[cond] = 2*S, b_up = -S       → up = 2*S*cond - S (threshold at 0.5)
    W_gate[a] = 1.0, b_gate = 0       → gate = a
    W_down = 1/S
    → Contributes: step(cond >= 0.5) * a

  Unit 1: (cancel pair)
    W_up[cond] = 2*S, b_up = -2*S     → up = 2*S*cond - 2*S (threshold at 1.0)
    W_gate[a] = -1.0, b_gate = 0      → gate = -a
    W_down = 1/S
    → Cancels over-activation for cond > 1

Units 2-3: Conditional path for 'b' - step(cond < 0.5) * b
  Unit 2:
    W_up[cond] = -2*S, b_up = S       → up = -2*S*cond + S (threshold at 0.5)
    W_gate[b] = 1.0, b_gate = 0       → gate = b
    W_down = 1/S
    → Contributes: step(cond < 0.5) * b

  Unit 3: (cancel pair)
    W_up[cond] = -2*S, b_up = 0       → up = -2*S*cond (threshold at 0.0)
    W_gate[b] = -1.0, b_gate = 0      → gate = -b
    W_down = 1/S
    → Cancels over-activation for cond < 0

IF_THEN Operation (emit_if_then):
---------------------------------
Implements: out = out + step(cond >= 0.5) * (a - out)
          = out * (1 - step(cond >= 0.5)) + a * step(cond >= 0.5)

Units 0-1: Add 'a' if condition is true
  Unit 0:
    W_up[cond] = 2*S, b_up = -S       → step(cond >= 0.5)
    W_gate[a] = 1.0, b_gate = 0       → multiply by a
    W_down = 1/S

  Unit 1: (cancel pair)
    W_up[cond] = 2*S, b_up = -2*S
    W_gate[a] = -1.0, b_gate = 0
    W_down = 1/S

Units 2-3: Subtract 'out' if condition is true
  Unit 2:
    W_up[cond] = 2*S, b_up = -S       → step(cond >= 0.5)
    W_gate[out] = -1.0, b_gate = 0    → multiply by -out
    W_down = 1/S

  Unit 3: (cancel pair)
    W_up[cond] = 2*S, b_up = -2*S
    W_gate[out] = 1.0, b_gate = 0
    W_down = 1/S

Result: When cond=1, adds a and subtracts out → final = out + a - out = a ✓
        When cond=0, adds nothing → final = out (unchanged) ✓

==============================================================================
KEY TECHNIQUES
==============================================================================

1. **Fractional Threshold with 2x Scaling**:
   - step(cond >= 0.5) implemented as step(2*cond >= 1)
   - Scales the fractional threshold (0.5) to an integer (1)
   - W_up = 2*S to double the input, b_up = -S for threshold at 1

2. **Cancel Pair Pattern**:
   - Every step function uses two units with opposite signs
   - Unit 0: positive contribution for values above threshold
   - Unit 1: negative contribution to cancel over-saturation
   - Ensures output stays in valid range [0, 1] for the step function

3. **Multiplication via W_gate**:
   - Step function computed in W_up (with b_gate = 0)
   - Value to multiply passed through W_gate
   - hidden = silu(up) * gate combines step function with value
   - Allows conditional selection of values

4. **Additive Residual Connection**:
   - All operations use residual: output = input + delta
   - IF_THEN exploits this to preserve initial value when cond=0
   - SELECT writes to new register, so initial value is 0

==============================================================================
DESIGN DECISIONS & LIMITATIONS
==============================================================================

Boolean Inputs Required:
------------------------
The conditional operations are designed for BOOLEAN conditions (0 or 1),
not analog values close to the threshold.

Why: The silu-based step function has a soft transition region around the
threshold. For inputs within ~0.3 of the threshold (0.5), the output will
not fully saturate to 0 or 1.

Examples:
  ✓ cond=0.0 → step=0.0 (works perfectly)
  ✓ cond=1.0 → step=1.0 (works perfectly)
  ✗ cond=0.2 → step≈0.4 (soft, not hard threshold)
  ✗ cond=0.8 → step≈0.6 (soft, not hard threshold)

For full saturation, the condition must be at least 0.5 away from the
threshold, which means using 0 or 1 (not intermediate values like 0.2, 0.8).

This is not a bug - it's an architectural characteristic. The operations
work correctly for their intended use case (boolean conditions).

Register Layout:
---------------
- SELECT uses 3 input registers (cond, a, b) and 1 output register
- IF_THEN uses 2 input registers (cond, a) and reads/writes 1 output register
- All registers must be physically distinct in the computation graph

==============================================================================
USAGE EXAMPLE
==============================================================================

In a computation graph, you can now express:

  # Conditional maximum: max(a, b) = (a > b) ? a : b
  cmp_node = CMP_GT(a, b)           # Returns 1 if a > b, else 0
  max_node = SELECT(cmp_node, a, b) # Select a if true, else b

  # Conditional clamp: if (x < 0) then x = 0
  cmp_node = CMP_LT(x, 0)           # Returns 1 if x < 0, else 0
  clamp_node = IF_THEN(cmp_node, 0) # Set x = 0 if true, else unchanged

These patterns enable complex control flow to be compiled into the neural
VM's FFN weight matrices.

## 🔍 Key Findings

### Stdlib Complexity
- Simple `int main() { return 42; }` compiles to **210 instructions** with stdlib
- Without stdlib: **6 instructions** (JSR, EXIT, ENT, IMM, LEV, LEV)
- Stdlib adds: memory management (malloc, free, memset, memcmp) initialization
- **Impact**: Makes testing slower and more complex

### L14 Bug Impact History
- Bug introduced in commit `831f298` (2026-04-08)
- Present for ~1 day before detection
- Affected ALL memory operations (JSR, ENT, PSH, SI, SC)
- Copy-paste error: address heads correctly use CLEAN_EMBED (for position encoding), value heads incorrectly copied this pattern

### Why CLEAN_EMBED vs OUTPUT Matters
- **CLEAN_EMBED**: Original token embedding (never changes during forward pass)
- **OUTPUT**: Layer output after FFN updates (changes as layers transform data)
- **Registers** (PC, AX, SP, BP, STACK0): Values stored in OUTPUT, not CLEAN_EMBED
- **L14 must read OUTPUT** to see register updates from L6 FFN

## 💡 Technical Insights

### Layer Data Flow for JSR
```
1. L6 FFN JSR: STACK0 = return_addr (writes to OUTPUT_LO/HI)
2. L14 Attn: Reads STACK0 from OUTPUT (NOT CLEAN_EMBED!) ← BUG WAS HERE
3. L14 Attn: Generates MEM token [MEM, new_sp_bytes, return_addr_bytes]
4. Shadow Memory: Stores mem[new_sp] = return_addr
5. L15 Attn: LEV reads from shadow memory → gets return_addr ✅
```

### Why Tests Pass Now
- **Before fix**: L14 read CLEAN_EMBED → MEM token had zeros → LEV read zeros ❌
- **After fix**: L14 reads OUTPUT → MEM token has return_addr → LEV reads correct value ✅

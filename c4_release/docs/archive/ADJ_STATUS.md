## Implementation Summary

ADJ (stack adjustment for function call cleanup) now executes entirely through neural weights.

### Neural Architecture

**L7 Head 1** - Operand Gather (line 4275-4291)
- Extended existing LEA head
- Fires on `OP_ADJ` flag
- Gathers SP → ALU staging dimensions
- Attends to `MARK_SP` marker

**L8 FFN** - Lo Nibble Addition (line 4516-4540)
- 256 units: exhaustive enumeration of nibble addition
- 120 units: carry detection (when a+b >= 16)
- Gates on `OP_ADJ` flag
- Outputs to `OUTPUT_LO` and `CARRY[0]`

**L9 FFN** - Hi Nibble Addition (line 4621-4639)
- 512 units: 256 with carry + 256 without carry
- Handles 4-byte signed addition
- Propagates carry from L8
- Outputs to `OUTPUT_HI` and `CARRY[1]`

**L6 FFN** - SP Writeback (line 3981-4000)
- 32 units: 16 lo nibble + 16 hi nibble
- Cancels identity carry-forward at SP marker
- Routes `AX_CARRY` → `OUTPUT` (SP register)
- Gates on `OP_ADJ && MARK_SP`

**Total**: 920 FFN units across 4 layers

### Pattern Used

Mirrors LEA implementation:
- **LEA**: `AX = BP + signed_immediate`
- **ADJ**: `SP = SP + signed_immediate`

Same ALU circuit, different register source/destination.

## Usage in C4 Programs

ADJ is **only used for function call cleanup**, not local variables.

### Example: Function Call

```c
int add(int a, int b) {
    return a + b;
}

int main() {
    return add(10, 32);  // Arguments pushed to stack
}
```

**Generated Bytecode:**
```
main:
    IMM 10        ; Load first argument
    PSH           ; Push to stack
    IMM 32        ; Load second argument
    PSH           ; Push to stack
    JSR add       ; Call function
    ADJ 16        ; Clean up 2 arguments (2 * 8 bytes)
    ...
```

ADJ adjusts SP by +16 to remove the 2 arguments (2 * 8 bytes per argument in 64-bit VM).

### Not Used For

Local variables use **ENT** (Enter function) for allocation:
```c
int func() {
    int a, b, c;  // Uses ENT with local_offset, NOT ADJ
    ...
}
```

## Impact on Handler Removal Plan

### Before ADJ Implementation
- 5 handlers remaining: ADJ, MALC, FREE, MSET, MCMP
- ENT blocked (needs ADJ for local allocation)

### After ADJ Implementation
- 4 handlers remaining: MALC, FREE, MSET, MCMP
- ENT still blocked (local allocation uses ENT's own SP adjustment, not ADJ)
- ADJ cleanup now pure neural

### Discovery
ENT does **not** use ADJ for local allocation. ENT has 4 steps:
1. Push old BP to stack (PSH-like)
2. BP = SP (register copy)
3. **SP -= local_offset** (direct adjustment in ENT, not ADJ call)
4. Memory write (needs MEM token)

ADJ is only for **function call cleanup** after JSR returns.

# Softmax-sharpness audit

Generated by `tests/test_softmax_sharpness.py` per Phase 1 Agent A
of `docs/ARCH_LEAKAGE_FIX_PLAN.md`. Audits every attention head
in the model returned by `compile_full_vm()` and measures how
sharp the head's softmax is when a synthetic context lights
its W_k gates at exactly one target K position.

- Threshold: mass on target >= **99.0%**
- Synthetic context length: 16
- Target K position: 4; Q position: 8
- Top gate dims per head (by |W| column-norm): 8

## Methodology and caveats

The probe uses `|W_q|` / `|W_k|` column-magnitude to identify each head's 'gate' input dims, then constructs a synthetic context where the target K position has those K-gate dims set to 1.0 and the Q position has the head's Q-gate dims (minus any K-gate overlap) set to 1.0. The score reported is the actual softmax mass the head places on the target position under this synthetic input.

**Caveats** — interpret failing heads carefully:

- Some heads use **negative** W_q weights at certain input dims to BLOCK firing on specific patterns (e.g. `MARK_PC_BLOCK = -S*50` is a common 'don't fire at PC-marker positions' pattern). Lighting such a dim at the Q position drives the score negative — the head fails the synthetic probe but is correct in real contexts where that dim is dark. A very negative `s_target` is a tell-tale sign.

- Heads keyed on dims OUTSIDE the top-N column-norm set may have their actual routing signal underrepresented in the synthetic context. Increase `TOP_GATE_DIMS` and re-run if a specific head looks suspicious.

- For the audit's intended purpose (catching softmax-leakage in the L4-L8 head zone identified in `ARCH_LEAKAGE_FIX_PLAN.md`), the useful signal is heads with **positive `s_target`** but low **mass@target** — those are the heads that see their target K but fail to drown the distractors. They are the primary Phase 2 D candidates.
